# コードの方針

- Test Driven Development (TDD)に厳密に則り、機能の追加の際は必ずモック・ドライバ・スタブも用意して外部依存無しで動作するユニットテスト及び外部依存有りで動作する統合テストを走らせて確実にすべてのテストにパスすること
- ユニットテストにおけるコードカバレッジリポートを生成し90%以上のカバレッジであることを保証すること。Rustの場合cargo-llvm-covを使う。
- 静的コードスキャン(rustの場合はcargo clippy --pedantic, pythonの場合Ruff)を利用しciriticalまたはhighなバイオレーションは修正すること。Lowまたはmediumのバイオレーションは許容する。
- 外部依存のあるロジックは必ずtraitを定義してモック・スタブを導入できるようにすること
- データのバリデーションはRustの場合new typeを使って実装すること
- 依存するライブラリのバージョンはできるだけ最新のものを使うこと

# ドキュメントの方針

- 以下のドキュメントをdocs以下に生成する
	- design.md ... 設計の詳細ドキュメント
	- api.md ... バックエンドAPIの各エンドポイントの仕様
	- dependencies.html ... フロントエンドとバックエンドAPIおよび共有コンポーネントが依存しているライブラリ名とバージョンそしてそのライブラリの簡潔な説明
- コードカバレッジレポートをhtml形式でdocs以下に生成する
-  ドキュメントやコードをコミットするときのメッセージがすべてUTF-8であること。Windowsでは時々Shift-JISだったりCP932だったりするので注意。

# 指示の分析と計画

- 主要なタスクを簡潔に要約してください。
- 記載された技術スタックを確認し、その制約内での実装方法を検討してください。
- 潜在的な課題とセキュリティに関する課題をリストアップしてください。
- タスク実行のための具体的なステップを詳細に列挙してください。
- それらのステップの最適な実行順序を決定してください。
- 必要となる可能性のあるツールやリソースを考慮してください。

このセクションは後続のプロセス全体を導くものなので、時間をかけてでも、十分に詳細かつ包括的な分析を行ってください。

# タスクの実行

- 特定したステップを１つずつ実行してください。
- 各ステップ完了後、簡潔に進捗を報告してください。
- 実行中に問題や疑問が生じた場合は即座に報告し、対応策を提案してください。
- 機能を実装する際は必ずユニットテスト及び統合テストを実行しコードカバレッジ及び全テストがパスしたことを確認してください。

# 品質管理

- 各タスクの実行結果を迅速に検証してください。
- エラーや不具合を発見した場合は、直ちに修正アクションを実施してください。
- コマンドを実行する場合は、必ず標準出力を確認し、結果を報告してください。

# 最終確認

- すべておタスクが完了したら、成果物全体を評価してください。
- 当初の指示内容との整合性を確認し、必要に応じて対応策を提案してください。
- ユニットテストのコードカバレッジが90%以上であること、リントエラーがないこと、コードフォーマッターが適用済みであるいことを確認してください。

# 重要な注意事項

- 不明点がある場合は、作業開始前に必ず確認を取ってください。
- 重要な判断が必要な場合は、その都度報告し、承認を得てください。
- 予期せぬ問題が発生した場合は、即座に報告し、対応策を提案してください。

# ショートカットエイリアス

- `/ask`: ユーザーがポリシーに関する相談を求めている場合。多角的な分析を含む積極的な回答を提供してください。明確な指示がない限り、相談中にタスクを実行しないでください。
- `/plan`: 作業計画を明確かつ詳細に概説し、相違点がないことを確認してください。合意に達した後にのみ実行に移ってください。
- `/debug`: バグの根本原因を特定します。5?7つの可能性のある原因をリストアップし、1?2つに絞り込みます。修正を適用する前に、ログを使用して仮説を検証してください。
- `/cmt`: コードの意図を明確にするために、適切なコメントとドキュメントを追加します。既存のコード形式に従ってください。
- `/log`: 適切なログレベルを考慮し、必要な情報のみを記録します。ログは簡潔に設計し、冗長性を避けてください。既存のコード形式に従ってください。
- `/talk_sanomaru`: さのまるとのカジュアルな対話用コマンドです。
- `/talk_kobaton`: コバトンとのカジュアルな対話用コマンドです。
- `/talk_fukka`: ふっかちゃんとのカジュアルな対話用コマンドです。
